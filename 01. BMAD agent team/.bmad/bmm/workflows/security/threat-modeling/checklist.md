# Threat Modeling Validation Checklist

## Scope Definition
- [ ] System clearly defined
- [ ] Assets identified
- [ ] Actors documented
- [ ] Trust boundaries marked

## Data Flow Diagram
- [ ] All components shown
- [ ] Data flows mapped
- [ ] Trust boundaries visible
- [ ] External entities included

## STRIDE Coverage
- [ ] Spoofing assessed
- [ ] Tampering assessed
- [ ] Repudiation assessed
- [ ] Information Disclosure assessed
- [ ] Denial of Service assessed
- [ ] Elevation of Privilege assessed

## Risk Assessment
- [ ] DREAD scores calculated
- [ ] Priorities assigned
- [ ] Attack scenarios described
- [ ] Current controls documented

## Mitigations
- [ ] Each threat has mitigation
- [ ] Control types specified
- [ ] Residual risk noted
- [ ] Implementation effort estimated

## Documentation
- [ ] Executive summary accurate
- [ ] Roadmap prioritized
- [ ] Assumptions documented
- [ ] Out of scope noted
