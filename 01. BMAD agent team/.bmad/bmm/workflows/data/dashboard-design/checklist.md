# Dashboard Design Validation Checklist

## Purpose and Audience
- [ ] Primary audience identified
- [ ] Key decisions documented
- [ ] View frequency established
- [ ] Viewing context considered

## Metrics Selection
- [ ] 5-9 metrics maximum
- [ ] All metrics actionable
- [ ] Targets/benchmarks included
- [ ] No vanity metrics
- [ ] Formulas documented

## Visualization Design
- [ ] Chart types appropriate
- [ ] Color usage meaningful
- [ ] Context provided (benchmarks)
- [ ] Labels clear
- [ ] Y-axis starts at zero (bars)

## Layout
- [ ] Most important top-left
- [ ] Logical grouping
- [ ] Information hierarchy clear
- [ ] White space appropriate
- [ ] Responsive design considered

## Interactivity
- [ ] Date filter included
- [ ] Key dimension filters
- [ ] Drill-down paths defined
- [ ] Conditional formatting specified
- [ ] Tooltips documented

## Data Requirements
- [ ] Sources identified
- [ ] Refresh frequency defined
- [ ] Performance considered
- [ ] Data quality assessed

## Report Quality
- [ ] Wireframe included
- [ ] Specifications complete
- [ ] Implementation notes clear
- [ ] Maintenance plan defined
